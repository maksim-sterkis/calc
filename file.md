# Codebase Architecture

The codebase is modular and component-based. Here is a breakdown of what each file is responsible for:

### 1. `main.cpp`
The main entry point of the application. It contains the interactive command-line REPL (Read-Eval-Print Loop) UI, handles user input/output, and manages the primary execution cycle.

### 2. `Config.hpp` / `Config.cpp`
Contains global settings and helper utilities.
* `USE_UNICODE`: Toggles whether to print raw unicode math symbols (like `√` and `π`) or terminal-safe equivalents (`root` and `pi`).
* Math utilities like `custom_gcd`, `custom_lcm`, and a registry of known function names (`is_function`).

### 3. `Token.hpp` / `AST.hpp`
These define the core data structures used during the parsing phase.
* **Token**: Represents a single "word" or symbol from the user's input (e.g., `NUMBER`, `PLUS`, `IDENTIFIER`).
* **ASTNode**: Represents a node in the Abstract Syntax Tree, which dictates the mathematical order of operations.

### 4. `Tokenizer.hpp` / `Tokenizer.cpp`
Responsible for reading the raw string inputted by the user (e.g. `2x + 5`) and breaking it down into a linear list of `Token` objects. It features a custom **Alphabetic Splitter** to protect keywords like `sin` and `derivative` while intelligently breaking apart adjacent variables for implicit multiplication (translating `xy` into `x * y`).

### 5. `Parser.hpp` / `Parser.cpp`
Implements a **Pratt Parser** (Top-Down Operator Precedence parser). It takes the linear list of `Token`s from the Tokenizer and constructs a hierarchical Abstract Syntax Tree (`ASTNode`), respecting mathematical rules like PEMDAS/BODMAS.

### 6. `ExactValue.hpp` / `ExactValue.cpp`
The mathematical heart of the CAS. It defines `ExactTerm` and `ExactValue`, which represent algebraic polynomials (e.g., `3x^2 * √5`). It contains the highly complex `simplify()` algorithms to combine like-terms, handle imaginary numbers (`i`), and reduce fractions.

### 7. `Engine.hpp` / `Engine.cpp`
The Algebraic Polynomial Engine. It contains the mathematical operators that act upon `ExactValue` objects:
* `add()`, `subtract()`, `multiply()`, `divide()`, `power()`, `compute_root()`, and `factorial()`.
* It also contains the `get_rational_form()` logic used for complex division simplification and output formatting.

### 8. `Evaluator.hpp` / `Evaluator.cpp`
The execution layer. It recursively walks through the Abstract Syntax Tree generated by the parser and "solves" it.
* Computes trigonometric functions (`sin`, `cos`, `tan`, etc.) and their inverses natively.
* Pre-processes identities (like logarithmic `ln(a) - ln(b) -> ln(a/b)`).
* Implements the `solve()` function for isolating variables in equations (linear, quadratic, radical, exponential, trig substitutions, and Durand-Kerner numerical roots for higher-degree polynomials).
* Implements the `limit()` function using L'Hôpital's rule.

### 9. `Calculus.hpp` / `Calculus.cpp`
The Symbolic Calculus engine.
* Dynamically traverses and intercepts the AST to calculate true mathematical derivatives (`differentiate_ast`) using power, product, quotient, and chain rules.
* Performs reverse-power rule analytical integration on polynomials (`integrate_polynomial`).

### 10. `Polynomial.hpp` / `Polynomial.cpp`
The Multivariate Algebraic Geometry engine. 
* Converts `ExactValue` equations into strict multivariate `Polynomial` objects.
* Implements **Buchberger's Algorithm** to compute Gröbner Bases, which is used to decouple and solve complex non-linear systems of polynomial equations.

### 11. `Matrix.hpp` / `Matrix.cpp`
The Linear Algebra engine.
* Defines matrix objects containing `ExactValue`s.
* Implements algorithms to compute the Determinant (`det`), Invert matrices (`invert`), and reduce matrices into Reduced Row Echelon Form (RREF) for solving linear systems of equations.

### 12. `Diagnostics.hpp` / `Diagnostics.cpp`
A lightweight error-handling module used to print human-readable syntax errors to the user when they type an invalid equation.
